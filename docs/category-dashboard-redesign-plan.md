# Category Dashboard Redesign Plan

Goal: make every section feel like a purpose-built operating surface, not a repeated page of cards.

## Phase 1 - Ship a Unique Dashboard Grammar For Every Personal Category

- [x] Vessel: body cockpit with readiness rings, training/nutrition/recovery lanes, and a next-session action rail.
- [x] Identity: compass dashboard with self-alignment, mission, blocker, and decision-frame zones.
- [x] Systems: command board with open-loop triage, automation posture, and closure lanes.
- [x] Ventures: portfolio radar with priority compression, blocker visibility, ROI/source cues, and strategy actions.
- [x] Career: leverage ladder with proof packaging, compensation path, credential path, and readiness actions.
- [x] Wealth: capital cockpit with engine map, visibility gap, priority allocation, and finance-source honesty.
- [x] Education: study runway with program anchor, checkpoint gap, overload watch, and course-action lane.
- [x] Knowledge: model forge with extraction queue, reference hierarchy, recency gaps, and action conversion.
- [x] Relationships: care map with privacy-first signals, touchpoint lane, environment fit, and next care action.

## Phase 2 - Deepen Each Category With Better Source Models

- [ ] Vessel: connect direct workout, nutrition, sleep/recovery recency and calculate readiness from real inputs.
- [ ] Identity: extract active mission, ideal-self gap, blockers, and principles into structured fields.
- [ ] Systems: expose real open loops, blocked tasks, automations, and closure rate from source records.
- [ ] Ventures: rank venture lines by momentum, upside, urgency, cost, and next decision.
- [ ] Career: ingest resume/project/outreach/interview readiness signals.
- [ ] Wealth: add live or manually maintained balance, obligations, income, runway, and allocation data.
- [ ] Education: add courses, due dates, assignment status, and study windows.
- [ ] Knowledge: add latest notes, references, books, models, and decision links.
- [ ] Relationships: keep raw detail private while surfacing safe reminders, cadence, and care actions.

## Phase 3 - Interaction Layer

- [ ] Add section-specific controls rather than generic filters: timeline scrubber for Vessel, compass mode for Identity, kanban/triage mode for Systems, portfolio sort for Ventures, ladder/proof mode for Career, allocation mode for Wealth, runway mode for Education, model browser for Knowledge, care cadence mode for Relationships.
- [ ] Make command palette actions section-aware so a button can create the right next move for that dashboard.
- [ ] Add empty and stale states that explain what source is missing without making the UI feel broken.

## Phase 4 - QA And Polish

- [ ] Check desktop, tablet, and mobile screenshots for each category.
- [ ] Verify text never overflows buttons, rails, or dense panels.
- [ ] Verify keyboard focus states and tap targets.
- [ ] Deploy and validate production URLs return 200.
